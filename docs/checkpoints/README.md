# Checkpoints Tecnicos

Diretorio de checkpoints versionados do ApplyFlow.

## Objetivo

Registrar estado real do sistema por data, com evidencias tecnicas auditaveis:

- escopo executado;
- riscos e decisoes;
- evidencias de teste/build/runtime;
- pendencias para proximo ciclo.

## Convencao de nome

- `YYYY-MM-DD-descricao-curta.md`

## Regras de registro

- Nao declarar validacao sem evidencia executada.
- Separar claramente:
  - `VALIDADO`
  - `NAO VALIDADO`
- Sempre indicar comandos executados e resultado observado.
